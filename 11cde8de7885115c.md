# ResourceBridge

ResourceBridge 是一个面向技术开发者与内容研究者的结构化外链与知识资源汇总系统。该项目并非传统的代码库或框架，而是一个精心编排的综合性技术百科文章索引门户，旨在解决互联网碎片化阅读时代优质技术内容难以系统化检索与沉淀的问题。项目通过对特定领域知识文章进行逻辑归集与导航重构，帮助目标用户即软件工程师、运维人员、技术博主以及计算机科学学习者，能够以目录化、可追溯的方式访问深藏在长尾站点中的高质量技术解析文档。

该项目当前处于第 8 批次建设阶段，整体规划涵盖 56 个构建批次，累计整合超过 180 个外部技术资源链接。ResourceBridge 不生产原始内容，而是通过人工与自动化工具结合的方式，对分散在互联网各处的技术百科类文章进行元数据提取、分类标引与可用性校验，最终形成一份可供本地部署或在线查阅的静态知识导航站点。项目核心价值在于将无序的网络文章转化为结构化的可查询资源库，显著降低开发者获取特定技术细节的时间成本。

## 功能概览

**批量资源导入解析** 支持通过脚本批量摄入外部文章链接，自动解析 URL 结构中的分类与编号信息，生成内部资源映射表。

**多维度分类导航** 根据文章所属技术领域、内容形态与目标读者层次，将资源划分为基础理论、工程实践、故障排查、性能调优等一级类目。

**本地全文检索** 集成轻量级全文索引引擎，支持对已收录文章标题与摘要进行关键词检索，并返回相关度排序结果。

**资源可用性监控** 定时检测已收录外链的 HTTP 响应状态码，自动标记失效链接并生成告警日志，保障导航数据的有效性。

**结构化数据导出** 支持将资源索引以 JSON、CSV 或 Markdown 表格格式导出，便于嵌入其他文档系统或进行二次数据分析。

**静态站点生成** 内置模板渲染引擎，可将资源列表与分类信息输出为完整的静态 HTML 站点，适用于 GitHub Pages 或 Nginx 部署。

**标签与关联推荐** 基于文章内容特征提取标签向量，在详情页自动展示同标签或同系列的其他关联文章入口。

## 应用场景

技术团队内部知识库建设。团队技术负责人可利用 ResourceBridge 将分散在团队收藏夹、个人博客与外部百科中的参考文章统一纳管，形成团队共用的技术查考手册，新员工入职时可快速通过该导航了解项目所涉及技术栈的周边知识脉络。

个人技术阅读流管理。独立开发者或学生可使用该项目维护个人的技术阅读清单，按周或按月导入新发现的优质文章，并通过分类导航避免遗忘或重复收藏，构建可持续积累的个人知识体系。

技术社区内容聚合展示。技术社区运营方可部署 ResourceBridge 作为社区的内容推荐墙，将社区成员发布的优秀技术博文与站外引用资料混合编排，为社区访客提供一站式的学习入口，提升社区内容资产的复用效率。

自动化资源备份与迁移辅助。在需要将文章链接从一个平台迁移至另一个平台时，ResourceBridge 的结构化索引表可作为映射依据，配合脚本批量生成带分类标签的迁移清单，减少人工整理遗漏。

## 快速开始

以下步骤适用于 Linux 与 macOS 环境，Windows 用户建议通过 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge

# 安装依赖（Python 3.10+ 与 pip）
pip install -r requirements.txt

# 初始化本地数据库并导入示例资源
python scripts/init_db.py
python scripts/import_resources.py --batch 8 --source data/batch_8_urls.txt

# 启动本地开发服务器
python app.py --host 127.0.0.1 --port 8080
```

执行完毕后，使用浏览器访问 http://127.0.0.1:8080 即可查看导航首页。如需构建静态站点，可运行 `python build_static.py --output ./dist`。

## 常见问题

问：导入资源时提示 URL 格式不合法，如何解决？

答：请检查待导入的 URL 是否包含协议头。当前导入器要求每条链接必须以 http:// 或 https:// 开头，且不包含换行符或多余空格。若原始链接为裸域名，请手动补全协议头后再执行导入。此外，部分链接包含大小写混合的扩展名如 .sHtML，此格式属于合法 URL 路径，导入器已做兼容处理，无需修改。

问：如何自定义分类规则，使特定域名下的文章自动归入指定类目？

答：编辑 `data/categories.yaml` 文件，在 `domain_rules` 节点下增加域名与类目的映射条目。例如，若希望将 `m.baike.occzl.cn` 下的所有文章归入 "百科综述" 类目，可添加 `m.baike.occzl.cn: "百科综述"`。修改后重新运行导入脚本即可生效，已有资源的分类不会自动更新，需使用 `--update` 参数强制刷新。

问：静态站点生成后，部分链接显示为失效状态，但浏览器中可正常访问，原因是什么？

答：可用性检测模块默认使用 HEAD 请求探测链接状态，部分服务器对 HEAD 请求返回 405 或 403，但允许 GET 请求。此为已知限制，可在 `src/crawler/validator.py` 中将检测方法切换为 GET 并设置超时参数。建议在低峰时段执行全量检测，避免对目标服务器造成压力。

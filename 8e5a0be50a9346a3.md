# Weblink Atlas

Weblink Atlas 是一个面向技术研究人员、文档维护者与内容聚合者的结构化外链资源管理工具。该项目并非一个传统的内容爬虫或链接短链服务，而是一套用于对大量分散的百科类、知识库类外链进行规范化收录、元数据标注与状态监控的静态资源索引系统。

项目定位为“技术资源外链汇总站”，其核心目标用户包括：需要维护大量外部参考链接的技术文档工程师、从事信息整合的开源知识库维护者、以及对特定领域百科条目进行批量采集与分析的数据研究员。Weblink Atlas 不提供代理、翻墙或内容抓取功能，仅作为链接的整理、分类与健康度检查前端，帮助用户在信息过载的环境中高效管理高价值外部资源。

通过 Weblink Atlas，用户可以快速建立自己的外链目录，对链接进行标签化分类，并通过内置的链接可用性检测机制，及时发现失效或变更的资源。项目采用纯静态方案生成，无需后端数据库，所有数据通过结构化 Markdown 与 JSON 文件维护，便于版本控制与协作编辑。

## 功能概览

**批量链接收录**：支持一次性导入大批量 URL 列表，自动解析域名与路径结构，生成标准化的资源条目。

**多级分类标签**：允许用户为每条链接添加自定义分类标签与优先级标记，便于后续按主题筛选与排序。

**链接状态检测**：内置轻量级 HTTP 状态检查器，可定期或手动触发对已收录链接的可用性验证，标记失效或重定向资源。

**元数据扩展字段**：每条链接可附加收录日期、摘要描述、原始来源站点、关联关键词等扩展信息，丰富资源维度。

**静态站点生成**：基于配置文件与资源列表，一键生成完整的静态 HTML 目录站点，可直接部署至任意 Web 服务器或 CDN。

**导入导出兼容**：支持 CSV、JSON 与纯文本列表格式的导入导出，便于与其他数据处理工具或脚本进行集成。

## 应用场景

**技术文档参考文献管理**：技术团队在撰写系统设计文档或 API 说明时，往往需要引用大量外部百科或规范文档。Weblink Atlas 可作为参考文献链接的统一登记处，确保所有引用可追溯、可验证。

**开源项目资源页构建**：开源项目维护者可使用 Weblink Atlas 快速生成项目的“相关资源”或“友情链接”页面，将分散在 README 或 Issues 中的外链集中展示，提升项目文档的专业度。

**信息爬虫前置链接池维护**：在进行定向数据采集前，研究人员可先用 Weblink Atlas 整理目标链接池，过滤重复与无效条目，并为每条链接标注采集优先级与频率，提高爬虫效率。

**知识库外链审计**：企业或组织内部的知识库系统常包含大量外部链接。通过 Weblink Atlas 的批量检测功能，可定期审计所有外链的有效性，及时更新或移除失效链接，保障知识库质量。

## 快速开始

以下操作步骤适用于 Linux 与 macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/weblink-atlas/weblink-atlas.git

# 进入项目目录
cd weblink-atlas

# 安装依赖（Python 3.8+ 与 pip）
pip install -r requirements.txt

# 运行初始化构建，生成静态站点到 _build 目录
python build.py --input data/links.json --output _build
```

构建完成后，打开 `_build/index.html` 即可浏览本地的外链索引页面。若需自定义收录列表，请编辑 `data/links.json` 或 `data/links.csv` 文件，然后重新运行构建命令。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Python | 3.8 或更高 | 用于运行构建脚本与状态检测工具 |
| pip | 20.0 或更高 | Python 包管理工具，用于安装依赖库 |
| requests | 2.25.0 或更高 | 用于发送 HTTP 请求以检测链接可用性 |
| markdown | 3.3.0 或更高 | 用于将 Markdown 格式的摘要渲染为 HTML |
| PyYAML | 5.4.0 或更高 | 用于解析 YAML 格式的配置文件与元数据 |
| jinja2 | 3.0.0 或更高 | 模板引擎，用于生成静态 HTML 页面 |
| click | 8.0.0 或更高 | 命令行交互框架，用于解析构建命令参数 |

所有依赖均可通过 `pip install -r requirements.txt` 一次性完成安装。建议在虚拟环境中进行部署，以避免与其他 Python 项目产生包版本冲突。

## 项目结构

```
weblink-atlas/
├── build.py                 # 主构建脚本，负责读取数据并生成静态站点
├── config.yaml              # 全局配置文件，包含站点标题、主题、检测参数等
├── requirements.txt         # Python 依赖列表
├── README.md                # 项目说明文档（本文件）
├── LICENSE                  # MIT 许可证文本
│
├── data/                    # 数据目录，存放所有链接与元数据
│   ├── links.json           # 主链接库，JSON 格式，包含全部收录条目
│   ├── links.csv            # 链接库的 CSV 导出，便于电子表格编辑
│   └── tags.yaml            # 分类标签定义与颜色映射
│
├── src/                     # 源代码目录
│   ├── core/                # 核心功能模块
│   │   ├── loader.py        # 数据加载器，支持 JSON/CSV 解析
│   │   ├── checker.py       # 链接状态检测器，基于 requests 实现
│   │   └── renderer.py      # 渲染引擎，调用 Jinja2 生成 HTML
│   ├── templates/           # Jinja2 模板文件
│   │   ├── base.html        # 基础布局模板
│   │   ├── index.html       # 首页列表模板
│   │   └── detail.html      # 单条链接详情页模板
```

## 贡献指南

1. 在 GitHub 上 Fork 本仓库，并在本地克隆您自己的副本。创建新的功能分支时，请使用 `feature/` 或 `fix/` 前缀，例如 `feature/batch-import-csv`。

2. 对数据文件（`data/links.json` 或 `data/links.csv`）进行新增、修改或删除操作时，请确保每一条目均符合 `docs/link-spec.md` 中定义的字段规范，包括必填字段（url、title、category）与可选字段（description、tags、priority）的完整性。

3. 若涉及代码修改（如 `src/` 目录下的 Python 文件），请一并更新对应的单元测试（位于 `tests/` 目录），并确保所有测试用例通过。运行测试命令：`pytest tests/`。

4. 提交 Pull Request 之前，请在本地执行一次完整的构建流程（`python build.py --input data/links.json --output _build`），确保生成的静态站点无报错，且页面展示与实际数据一致。

5. 提交 PR 时，请填写清晰的标题与描述，说明本次变更的目的、影响范围以及测试情况。若为新增功能，请补充对应的文档更新。

## 常见问题

**问：收录的链接数量很大，构建速度会变慢吗？**

答：对于 5000 条以下的链接，构建时间通常在 10 秒以内。主要耗时在于链接状态检测阶段，该阶段默认启用并发请求（并发数可配置）。若仅需生成静态页面而不进行实时检测，可在构建时添加 `--skip-check` 参数跳过检测步骤，大幅提升构建速度。

**问：我可以用 Weblink Atlas 管理内网或需要登录验证的链接吗？**

答：可以。链接检测模块支持自定义请求头（如 Cookie 或 Authorization Token），您可以在 `config.yaml` 的 `checker.headers` 字段中配置全局请求头。但请注意，该工具不处理复杂的登录流程（如 OAuth 或表单提交），仅支持静态请求头注入。对于需要动态会话的链接，建议将检测功能关闭，仅作为静态目录使用。

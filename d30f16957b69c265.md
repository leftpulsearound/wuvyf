# ResourceHub

ResourceHub 是一个面向开发者和技术研究人员的结构化技术资源导航与外链汇总工具。该项目不对托管内容进行二次编辑或转载，而是以目录化、标签化的方式对互联网上的高质量技术文章、百科条目和开发文档进行索引与分类，帮助用户在海量信息中快速定位到特定主题的原始资料。

ResourceHub 定位于中长尾技术知识的发现与回溯。当开发者需要查找某个冷门函数的使用细节、某类算法原理的通俗解释或特定框架的历史版本特性时，可通过该项目的分类索引和关键词映射直接定位到外部原文链接，避免通用搜索引擎返回大量低相关度结果。该项目本身不存储任何侵权内容，仅提供 URL 索引服务。

## 功能概览

结构化分类索引：按技术领域、语言生态、应用层级对收录的 URL 进行多级标签划分，支持按分类浏览。

全文元数据检索：对每个收录链接提取标题、关键词和摘要信息，构建本地检索索引，支持布尔查询和模糊匹配。

批量导入与去重：支持通过文本文件或标准输入流批量导入 URL 列表，自动检测重复条目并合并标签。

链接状态巡检：周期性对已收录链接进行 HTTP 状态检查，标记失效链接并生成巡检报告，便于维护者清理或更新。

自定义标签体系：允许用户为任意链接添加自定义标签（如 python、algorithm、security），构建个性化分类视图。

导出与集成：支持将分类结果导出为 JSON、CSV 或 Markdown 表格格式，便于嵌入技术文档、博客或内部知识库。

访问统计与热度排序：基于本地记录的外链点击次数生成热度排行，辅助用户识别社区关注度较高的资源。

## 应用场景

技术团队内部知识库沉淀：技术负责人可将 ResourceHub 部署为团队内部的技术文档索引面板，将分散在个人书签、即时通讯群组中的外部链接统一收录并分类，新成员入职时可直接通过该索引体系快速了解团队常用的技术栈与参考资源。

个人开发者学习路径管理：正在系统学习某个技术方向（如分布式系统、编译原理）的开发者，可将每日阅读的文章链接录入 ResourceHub，并通过标签标记阅读状态（如待读、已读、重点复读），形成可追溯的个人学习档案。

技术社区内容聚合与推荐：技术博客作者或社区运营人员可利用该项目的分类索引功能，将有价值的站外内容按主题组织为专题列表（如「Go 语言并发实战」「Kubernetes 故障排查案例」），以导航页形式分享给读者，降低优质内容的发现成本。

离线资源备份与失效预警：教育机构或企业培训部门可将课程涉及的参考资料链接统一纳入 ResourceHub 管理，利用链接状态巡检功能定期检测外链可用性，及时通知相关人员更新或替换失效教材链接。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，要求系统已安装 Git 与 Node.js（v18 及以上）。

```bash
git clone https://github.com/resource-hub/resource-hub.git
cd resource-hub
npm install
npm run build
npm start
```

执行完成后，服务默认监听 3000 端口。访问 http://localhost:3000 即可进入分类浏览界面。首次启动时系统会自动创建示例分类与占位数据，用户可通过管理后台或直接编辑 `/data/sources.json` 文件进行初始数据导入。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js | 是 | 运行时环境，要求 v18.0.0 或更高版本，推荐 LTS 版本 |
| npm | 是 | 包管理器，随 Node.js 一同安装，用于安装项目依赖 |
| SQLite3 | 是 | 嵌入式数据库，用于存储链接元数据和标签索引，无需额外安装服务进程 |

| curl 或 wget | 否 | 用于执行链接状态巡检时的 HTTP 探测，若缺失则使用 Node.js 原生 http 模块降级 |
| pm2 | 否 | 生产环境进程守护工具，推荐用于后台持续运行 |
| jq | 否 | 命令行 JSON 处理工具，仅在导出或脚本调试时提供便利 |

## 常见问题

Q: ResourceHub 是否存储或缓存外部链接的完整网页内容？
A: 否。ResourceHub 仅存储 URL 地址、用户自定义标签、标题摘要（从 HTML title 或 meta 标签提取）以及链接状态（如 HTTP 状态码）。项目不缓存页面正文、不保存图片等静态资源，所有内容访问均重定向至原始链接。用户点击条目时直接跳转至第三方页面，项目本身不承担任何内容审核责任。

Q: 如何批量导入大量 URL 并自动分类？
A: 项目提供了命令行导入工具 scripts/import-batch.js，支持从纯文本文件（每行一个 URL）或 JSON 数组格式导入。导入时系统会根据 URL 路径特征（如域名中的关键词、路径中的数字区间）应用默认分类规则，用户可在导入完成后通过管理界面或直接编辑数据库进行调整。对于包含数百个链接的批次，建议使用 --delay 参数控制请求间隔，避免触发源站的反爬策略。

Q: 链接状态巡检显示大量 404 错误，应如何处理？
A: 首先确认巡检时间窗口内目标网站是否可正常访问，部分站点可能有临时维护或频率限制。若确认链接已永久失效，可通过管理界面将该条目标记为「已失效」并归档，或使用「替换链接」功能更新为有效的替代 URL。项目也支持导出失效链接清单（JSON 格式），便于批量查找替代源或向源站管理员反馈。对于持续返回 5xx 错误的站点，建议降低巡检频率以避免对源站造成额外压力。

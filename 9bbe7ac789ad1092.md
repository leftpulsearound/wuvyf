# LinkPedia 技术资源导航

LinkPedia 是一个面向开发者、技术研究人员与开源项目维护者的结构化外链资源汇总平台。项目定位于技术文档、百科条目、开发教程与工程实践类高质量外部链接的集中整理与分类索引，帮助用户快速定位特定技术主题下的信息入口。

项目本身不生产内容，而是通过人工筛选与社区贡献相结合的方式，建立可维护的链接资源库，解决技术学习与开发过程中信息分散、检索效率低、优质来源难以发现等问题。LinkPedia 适用于个人知识管理、团队技术文档库建设、开源项目外部引用规范化等场景。

## 功能概览

**批量链接导入与去重** 支持从文本文件、CSV 或剪贴板批量导入 URL，自动检测重复条目并合并标签。

**自定义分类体系** 允许用户按技术领域、内容类型、来源站点等维度建立多级分类树，灵活组织资源。

**全文与元数据检索** 基于标题、描述、标签、域名等字段进行快速过滤，支持模糊匹配与精确查询。

**资源状态健康检查** 定期对已收录链接进行可达性检测，标记失效或重定向的条目，辅助维护。

**标签与评分系统** 用户可为资源添加自定义标签并打分，社区共识帮助新访客筛选高质量内容。

**导出与嵌入支持** 资源列表可导出为 JSON、Markdown 或 CSV 格式，并提供 iframe 嵌入片段供团队内部 Wiki 使用。

**访问统计与热度排序** 记录每个链接的点击次数与最近访问时间，支持按热度或更新时间排序展示。

## 应用场景

技术团队内部知识库建设。开发团队在项目迭代过程中积累大量外部参考资料，包括 API 文档、技术博客、解决方案讨论帖等。LinkPedia 提供统一入口，避免知识散落在聊天记录或浏览器收藏夹中。

开源项目 README 与文档的外部链接管理。开源项目维护者需要在文档中引用大量第三方资源，LinkPedia 可生成规范化的链接清单，确保 URL 格式统一且可追溯，同时便于定期校验链接有效性。

技术培训与新手引导。培训讲师或团队负责人可以为新入职人员预置学习路径，将入门教程、最佳实践、常见问题解答等资源分类整理，降低初始学习阶段的搜寻成本。

技术调研与竞品分析。研究人员在对某一技术领域进行横向对比时，可将候选项目、论文、评测报告等链接集中收录，通过标签和备注字段记录关键结论，形成可复用的调研素材库。

## 快速开始

以下命令将在本地克隆项目仓库、安装依赖并启动开发服务器。

```bash
git clone https://github.com/linkpedia/linkpedia.git
cd linkpedia
npm install
npm run dev
```

执行成功后，访问控制台输出的本地地址即可进入 LinkPedia 实例。默认使用 SQLite 作为存储后端，无需额外配置数据库服务。

## 安装要求

| 依赖 | 必需版本 | 说明 |
|------|----------|------|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用 nvm 管理版本 |
| npm | 9.x 或 10.x | 包管理器，随 Node.js 一同安装 |
| SQLite | 3.35 以上 | 默认嵌入式数据库，无需独立安装 |
| Git | 2.30 以上 | 用于克隆仓库和版本管理 |
| Chrome 或 Firefox | 最新稳定版 | 仅开发阶段用于测试前端界面 |

## 贡献指南

提交链接资源。通过 GitHub Issue 或 Pull Request 提供新的外链条目，需附带标题、原始 URL、所属分类及简要描述。提交前请使用项目提供的去重工具检查是否已存在。

完善分类体系。若现有分类无法覆盖某类资源，可在 Issue 中提出新增分类建议，说明分类名称、父级分类及涵盖范围，经维护者讨论后合并。

改进健康检查机制。链接可达性检测模块支持自定义超时时间与重试策略，欢迎提交针对不同站点类型（如国内/海外服务）的差异化检测参数优化方案。

补充文档与示例。用户手册、API 文档或部署指南中若存在表述不清或缺失部分，可提交 Markdown 格式的修订内容，并附上修改理由。

报告问题与建议。使用 GitHub Issues 提交缺陷报告或功能请求，需包含复现步骤、预期行为与实际行为对比，以及运行环境信息。

## 常见问题

部署后无法访问外部链接怎么办。

LinkPedia 自身不代理外部资源，仅存储和展示 URL。若运行环境存在网络策略限制（如防火墙或代理），需在系统层面配置 HTTP_PROXY 环境变量，或在 config/default.json 中设置 request 代理选项。健康检查功能同样遵循该代理配置。

如何迁移已有收藏夹或书签到 LinkPedia。

目前支持从 Chrome 书签导出文件（HTML 格式）和 Firefox 书签 JSON 导出文件进行导入。在管理后台的「导入」功能中选择对应格式并上传文件，系统将自动解析并呈现预览列表，确认后批量写入数据库。

数据库从 SQLite 迁移到 PostgreSQL 是否可行。

可以。项目使用 Sequelize ORM，支持多种数据库方言。需在 config/ 中修改 dialect 为 postgres，并配置对应的连接参数。注意部分 SQLite 特有的数据类型需要手动调整迁移脚本，建议在测试环境先行验证。

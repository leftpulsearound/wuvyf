# WapWiki Link Aggregator

WapWiki Link Aggregator 是一个面向技术文档研究者、知识库维护者和内容聚合平台运营者的轻量级外链资源归集系统。该项目以移动端百科类文章为主体数据源，提供结构化的 URL 整理、分类索引与快速检索能力，帮助用户从大量离散的链接中高效定位所需信息。

本项目定位于技术资源导航与知识外链管理工具，尤其适用于需要批量处理、分类标注和持续追踪移动端百科条目的场景。通过统一的入口，用户无需手动整理分散的链接，即可获得包含文章标识、分类建议和基础元信息的可操作数据集。

## 功能概览

**批量链接导入**：支持从文本文件、CSV 或直接粘贴的 URL 列表中批量导入文章链接，自动解析文章 ID 与来源域名。

**智能分类建议**：根据 URL 路径中的数字特征与常见百科词条模式，为每条链接推荐可能的分类标签，如技术、历史、文化、科学等。

**元信息抓取占位**：内置 HTTP 请求模板，可配置批量获取文章标题、摘要等基础元数据，为后续索引提供数据基础。

**链接状态检测**：定期检查已收录链接的可访问性，标记失效或重定向的条目，确保资源库的活跃度与可用性。

**多格式导出**：支持将整理后的链接列表导出为 JSON、CSV 或纯文本格式，便于集成到其他知识管理工具或静态站点生成器中。

**关键词检索**：基于文章 ID 与路径特征提供简单的关键词匹配检索，帮助用户在大量链接中快速筛选目标条目。

## 应用场景

技术文档团队的外链治理：技术文档团队在维护在线帮助中心或开发者门户时，常需要引用外部百科或技术文章。本项目可帮助团队统一管理这些外链，定期检查链接有效性，并在文档版本更新时快速定位引用来源。

知识库自动化构建：知识库管理员可利用本项目的批量导入与分类建议功能，将分散的百科文章链接快速归入不同主题类别，减少人工整理时间，提升知识库构建效率。

数据采集管道的前置处理：在构建数据采集或爬虫管道时，本项目可作为前置环节，对目标 URL 进行清洗、去重和初步分类，使采集任务更具针对性与可管理性。

个人研究者的资源归档：从事跨学科研究或个人项目的开发者，可使用本项目归档研究过程中收集的大量参考资料链接，通过统一接口进行检索和回溯。

## 快速开始

以下命令演示了从仓库克隆、安装依赖到运行基础链接导入流程的完整步骤。

```bash
git clone https://github.com/example/wapwiki-link-aggregator.git
cd wapwiki-link-aggregator
npm install
cp .env.example .env
# 编辑 .env 文件配置数据存储路径与抓取参数
node scripts/import-links.js --input ./data/raw_links.txt
node scripts/check-status.js --concurrency 10
node scripts/export-list.js --format json --output ./dist/links.json
```

│   └── integration/

## 常见问题

**问：导入链接时支持哪些输入格式？**

答：目前支持纯文本文件（每行一个 URL）、CSV 文件（需包含名为 url 的列）以及直接通过命令行参数传递的逗号分隔 URL 列表。后续版本将增加对 JSON 数组格式的支持。

**问：如何调整链接状态检测的并发数？**

答：您可以通过 `--concurrency` 参数或在 `.env` 文件中设置 `CHECK_CONCURRENCY` 变量来调整并发请求数量。默认值为 10，建议根据网络环境与目标服务器的承受能力适当调整，避免触发访问限制。

**问：分类建议的规则是否可以自定义？**

答：可以。分类规则基于 `src/core/classifier.js` 中的规则表实现。您可以根据需要修改或扩展该规则表，添加新的路径匹配模式与对应的分类标签。修改后建议运行分类相关的单元测试以确保规则正确生效。

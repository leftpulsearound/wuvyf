# WAP百科聚合索引系统

WAP百科聚合索引系统是一个面向移动端百科内容的高性能元数据采集与导航平台，专注于对多源百科站点进行结构化数据提取、分类索引与全文检索支持。本项目服务于内容聚合开发者、知识图谱构建团队以及移动端轻量级百科应用开发者，帮助其在不直接访问原始百科站点的情况下，通过统一的API和静态索引数据快速定位所需知识条目。

本项目定位为技术资源与外链汇总中间件，不对原始内容进行修改或重新发布，仅提供URL索引、分类标签、摘要快照和关联推荐等增值服务。通过定时增量抓取与存量校验机制，确保索引库与源站点保持逻辑一致性。

## 功能概览

**批量链接采集**：支持从指定入口页面递归或扁平化提取百科类文章链接，自动过滤非目标域名的外链。

**结构化字段解析**：对每篇百科文章提取标题、正文摘要、发布时间、分类标签、相关词条等元数据字段。

**多级分类索引**：根据文章内容自动打标或依据URL路径层级生成不少于三级的分类树，方便按主题浏览。

**全文检索接口**：基于倒排索引提供标题和摘要级别的关键词检索，支持布尔表达式与模糊匹配。

**增量更新机制**：记录每次采集的任务ID与时间戳，仅拉取自上次更新以来发生变更或新增的文章。

**去重与校验**：基于文章URL和内容哈希值双重去重，对已入库链接定期发起HEAD请求校验可用性。

**外链导出格式**：支持将索引结果导出为JSON、CSV和纯文本URL列表三种格式，适配不同下游系统。

**状态监控面板**：提供命令行下的任务进度视图和错误日志分级输出，便于运维人员跟踪采集任务健康度。

## 应用场景

作为百科类移动应用的后端数据准备工具。开发者在构建一个轻量级百科查询App时，无需自行维护庞大的百科数据库，而是通过本项目定期从指定站点拉取索引数据，在本地建立只读缓存，从而降低服务器成本和内容合规风险。

作为知识图谱实体链接的候选来源。知识图谱项目在实体消歧和关系补全阶段，可以利用本系统输出的带分类标签的URL列表，快速获取与特定实体相关的百科参考链接，用于人工审核或自动化爬取。

作为内部文档系统的外部参考资源挂载。企业内部的Wiki或技术文档平台可以集成本系统提供的分类索引，在相关技术词条页面自动推荐对应的外部百科链接，丰富参考资料维度。

作为学术研究的数据采样工具。社会科学或信息科学研究者需要采集特定领域百科文章的变化趋势时，可借助本系统的按分类导出功能，快速获得某一类别下的全量URL清单，用于后续的频率分析和内容比对。

## 快速开始

以下操作假设系统环境为Ubuntu 22.04 LTS或macOS 13.x，已安装Git和Python 3.10及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/example/wap-baike-indexer.git
cd wap-baike-indexer

# 安装项目依赖
pip install -r requirements.txt

# 执行首次全量采集任务（示例入口为默认配置）
python run_indexer.py --mode full --output ./data/index_$(date +%Y%m%d).json
```

首次运行会在`./data/`目录下生成索引文件，并在`./logs/`目录写入详细任务日志。若需自定义采集源，请修改`config/sources.yaml`中的入口配置。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 或更高 | 核心运行环境，用于采集脚本和索引构建 |
| requests | 2.31.0 或兼容版 | 发送HTTP请求，支持重试和超时控制 |
| beautifulsoup4 | 4.12.0 或兼容版 | 解析HTML文档，提取结构化字段 |
| lxml | 4.9.0 或兼容版 | 作为BeautifulSoup的解析器后端，提升解析速度 |
| redis | 6.0 或更高（可选） | 用于分布式去重锁和任务队列状态缓存，单机模式可禁用 |
| sqlite3 | 系统自带 | 存储采集任务记录、URL指纹和分类映射表 |
| git | 任意稳定版本 | 用于克隆仓库和管理补丁 |

## 项目结构

```
wap-baike-indexer/
├── config/                              # 系统配置目录
│   ├── sources.yaml                     # 采集源配置（入口URL、请求头模板）
│   ├── classifier_rules.yaml            # 分类打标规则（关键词到分类的映射）
│   └── logging.conf                     # 日志分级与轮转策略配置
├── core/                                # 核心采集与索引模块
│   ├── fetcher.py                       # 异步HTTP请求器，含重试与代理支持
│   ├── parser.py                        # HTML解析器，基于BeautifulSoup4实现
│   ├── extractor.py                     # 元数据字段提取逻辑（标题、摘要、标签）
│   └── indexer.py                       # 倒排索引构建与序列化
├── storage/                             # 存储与状态管理
│   ├── sqlite_store.py                  # SQLite任务记录与指纹存储
│   ├── redis_cache.py                   # Redis缓存连接与去重锁实现（可选）
│   └── file_exporter.py                 # 索引结果导出为JSON/CSV/TXT
├── cli/                                 # 命令行入口与交互
│   ├── run_indexer.py                   # 采集任务主入口，支持full/incremental模式
│   ├── query.py                         # 索引查询命令行工具
│   └── monitor.py                       # 任务状态与日志监控简易面板
├── tests/                               # 单元测试与集成测试
│   ├── test_fetcher.py                  # 网络请求模块测试（含mock）
│   ├── test_parser.py                   # 解析器对不同HTML结构的容错测试
│   └── test_indexer.py                  # 索引构建与查询正确性验证
├── docs/                                # 文档目录（详见文档导航）
│   ├── user_guide.md
│   ├── developer_guide.md
│   ├── api_reference.md
│   └── ops_manual.md
```

## 常见问题

问：采集任务运行时遇到HTTP 403或反爬验证如何解决？

答：项目在config/sources.yaml中提供了请求头模板字段，可自定义User-Agent和Referer。若目标站点启用更严格的反爬策略，建议将采集间隔调整至5秒以上，并启用代理池配置。对于依赖Cookie验证的站点，可在fetcher.py中扩展会话管理逻辑。本项目不提供绕过合法反爬措施的手段，使用者应遵守目标站点的robots.txt协议。

问：索引文件体积过大，检索响应变慢怎么办？

答：建议启用redis缓存并配置LRU淘汰策略，将热点查询结果缓存至内存。同时可以调整indexer.py中的分片大小参数，将单次加载的索引分片限制在50MB以内。对于长期运行的部署，可以配置每日定时任务，在低峰期重新构建紧凑索引并清理历史冗余数据。

问：如何仅采集某一特定分类下的文章？

答：可在运行采集任务时通过命令行参数--filter-category指定分类名称，系统会在解析阶段对文章进行打标，仅保留匹配分类的结果入库。若需更精细的过滤，可在config/classifier_rules.yaml中修改关键词映射表，或通过--url-pattern正则参数直接筛选URL路径。

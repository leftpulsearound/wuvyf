# LinkVault Resource Aggregator

LinkVault 是一个面向技术研究者、文档维护者和内容聚合者的外链资源归集与导航系统。该项目不对资源内容进行二次加工，而是提供结构化的索引、分类展示和本地快速启动环境，帮助用户在大量分散的外部链接中建立清晰的访问路径。LinkVault 适用于需要批量管理、分类展示或定期同步外部文章链接的个人或团队，尤其适合作为技术百科、知识库或资讯站点的底层链接支撑模块。

本项目第 14/56 批资源共计收录 180 个外部链接，全部来自 occzl.cn 域下的移动端百科文章。LinkVault 不承担外部内容的可用性验证责任，但提供完整的链接清单、本地运行环境以及可扩展的分类框架。

## 功能概览

**批量链接归集**：支持将大量原始 URL 按来源域名、路径模式自动归类，并提供去重与格式校验功能。

**多级分类索引**：允许用户为每个链接标注自定义标签（如「技术百科」「移动端」「长尾词」），并支持按标签筛选和排序。

**本地化快速预览**：通过内置的轻量级 Web 服务器，用户可在本地启动站点，以卡片或列表形式浏览所有资源链接，无需依赖外部数据库。

**链接状态检测**：提供可选的定时健康检查模块，对每个 URL 发送 HEAD 请求，记录响应状态码并生成可用性报告。

**导入导出兼容**：支持以 JSON、CSV 和纯文本格式导入链接列表，也支持将当前索引导出为 Markdown 表格或结构化数据文件。

**搜索与筛选**：基于 URL 关键字、文件名后缀（如 .sHtML）、路径数字段进行快速搜索，支持正则表达式高级过滤。

**访问统计记录**：记录每个链接在本地环境中的点击次数与最后访问时间，辅助用户判断资源热度。

**扩展字段支持**：为每个链接预留备注字段，用户可手动补充摘要、关键词或失效替代链接。

## 应用场景

技术百科站点维护人员可利用 LinkVault 定期归集移动端百科文章链接，将其作为站点侧边栏「相关阅读」或「延伸资料」的数据源，减少人工复制粘贴的工作量。

知识库建设团队在整理外部参考资料时，可使用 LinkVault 导入大量散落链接，按主题（如前端、后端、运维）打标签，并生成分类索引页，方便团队成员快速定位。

个人研究者或博主在撰写长文时，可将收集到的引用链接统一存入 LinkVault，利用搜索和筛选功能快速查找特定编号或路径的文章，避免反复切换浏览器标签页。

自动化运维脚本可将 LinkVault 的链接检测功能集成到 CI 流程中，每日检查外部百科链接是否可访问，并自动生成失效链接报告，供维护者更新。

## 快速开始

以下命令适用于 Linux / macOS / Windows（WSL 或 Git Bash）环境。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装依赖（基于 Node.js 22 LTS）
npm install

# 导入本批次资源列表（resources/batch_14.txt）
npm run import -- --file resources/batch_14.txt --source occzl

# 启动本地开发服务器
npm run dev
```

启动后，访问终端输出的本地地址（通常为 http://localhost:3000）即可浏览所有链接。如需执行一次性的健康检查，可运行：

```bash
npm run check:links
```

## 安装要求

| Node.js | 22.0.0 或更高 | 运行时环境，推荐使用 LTS 版本 |

| 操作系统 | Linux / macOS / Windows 10+ | 跨平台支持，Windows 推荐使用 WSL2 |
| 浏览器 | 现代浏览器（Chrome / Firefox / Edge） | 用于访问本地 Web 界面 |

## 常见问题

**Q：导入包含大量 URL 的文本文件时，是否支持自动去重和格式清洗？**

A：支持。导入模块会去除首尾空白字符，统一将协议头转换为小写，并基于完整 URL 字符串进行去重。重复条目会被记录到日志中但不会写入数据库。同时，系统会检测并提示明显格式错误的条目（如缺失协议头或包含非法字符）。

**Q：健康检查模块对目标服务器会造成压力吗？**

A：默认配置下，健康检查采用单线程顺序请求，且每个请求间隔不少于 200 毫秒。对于本批次 180 个链接，完整检测耗时约 2 至 3 分钟，并发度极低，不会对目标站点造成可感知的负载。用户可通过配置调整间隔时间和超时阈值。

**Q：是否可以自定义前端页面的品牌标识和配色？**

A：可以。所有前端静态资源位于 `src/client/static/` 目录，CSS 变量定义在 `theme.css` 中，用户可直接覆盖。同时支持通过 `config/default.yaml` 中的 `branding` 字段修改站点标题、页脚信息和 favicon 路径。

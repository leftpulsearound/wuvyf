# LinkPedia

LinkPedia 是一个面向开发者、技术研究人员与信息分析人员的结构化外链资源聚合平台。该项目不生产内容，而是对分散于互联网各处的优质百科类、技术参考类与数据类文章进行系统性采集、分类与索引，帮助用户快速定位特定主题下的高价值外部链接。LinkPedia 适合需要批量查阅技术文档、进行信息溯源或构建知识图谱的团队与个人，通过统一的条目格式与稳定的资源映射关系，降低信息检索的重复劳动成本。

## 功能概览

- 按主题分类的链接索引：所有收录链接按照技术领域、问题类型与内容形式进行粗粒度分类，支持快速筛选。

- 原始链接直出模式：每个资源条目均以原始 URL 形式呈现，不进行重定向、短链转换或协议变更，确保链接可追溯性。

- 本地化缓存标识：系统为每个外链生成内容摘要哈希与状态标记，便于用户判断链接是否可访问及内容更新情况。

- 多维度检索支持：基于链接文件名、目录结构与编号规则，提供基础的 grep 与正则表达式检索能力。

- 定期校验与失效标记：通过自动化脚本定期检测收录链接的可达性，并对失效链接进行标注。

- 批量导入与导出：支持通过文本文件批量新增链接条目，并可导出为 JSON 或 CSV 格式供下游工具使用。

- 静态站点生成兼容：项目目录结构设计为可直接挂载至静态站点生成器（如 Hugo、Jekyll），便于快速发布为在线导航站。

## 应用场景

1. 技术文档编写与素材引用：技术博主或文档作者在撰写教程时，可通过 LinkPedia 快速查找特定技术点的外部参考资料链接，直接作为引用来源。

2. 信息聚合与内容审核：内容运营团队可使用 LinkPedia 对大量外链进行统一整理与初步审核，确保外链指向内容与主题相关且来源可靠。

3. 学术研究与文献溯源：研究人员在进行文献综述或技术演化分析时，可利用本项目的链接清单进行批量访问与信息提取，节省逐一手动收集的时间。

4. 知识库构建的前置处理：企业知识管理团队可将 LinkPedia 作为外部数据源接入层，通过解析链接列表，持续丰富内部知识库的外部参考维度。

## 快速开始

以下命令演示如何将 LinkPedia 项目克隆至本地，安装基础依赖并启动本地预览服务。

```bash
git clone https://github.com/linkpedia/linkpedia.git
cd linkpedia
pip install -r requirements.txt
python scripts/serve.py --port 8080
```

如需仅获取链接清单，不启动服务，可执行：

```bash
python scripts/extract_links.py --input data/links.txt --output output/links.json
```

## 贡献指南

1. 在 GitHub 上 Fork 本仓库，并克隆至本地开发环境。新建一个以 `batch/` 为前缀的特性分支用于本次贡献。

2. 在 `data/links.txt` 末尾追加新增链接，每行一条，确保链接为完整的原始 URL。若需调整分类，同时修改 `data/categories.yaml` 中的映射关系。

3. 执行 `make validate` 运行校验脚本，检查新增链接的格式合法性及网络可达性。校验通过后，执行 `make test` 确保单元测试全部通过。

4. 提交变更时，使用规范的提交信息格式，例如 `add: 第8批链接，共150条` 或 `fix: 更正分类映射中的拼写错误`。提交前请确保未改动他人已提交的链接条目。

5. 向主仓库发起 Pull Request，并在描述中说明本次新增或修改的链接数量与分类调整原因。等待维护者审核并合并。

## 常见问题

Q: 链接无法访问怎么办？
A: 项目提供了自动校验脚本 `scripts/validator.py`，可定期运行检测所有链接的状态。若发现单个链接失效，可在 `data/links.txt` 中将其注释或移除，并提交变更说明。若为临时性不可达，建议等待一段时间后重新校验。

Q: 如何批量导入其他来源的链接？
A: 将其他来源的链接整理为每行一个 URL 的纯文本文件，然后使用 `scripts/import_links.py --source your_file.txt --append` 命令追加至 `data/links.txt`。追加后请手动检查分类映射是否需要同步更新。

Q: 为什么链接 URL 不做任何规范化处理？
A: LinkPedia 的设计原则之一是保留用户提供的原始链接形态，包括协议类型、域名格式及大小写。这保证了链接的可追溯性与原始意图，避免因自动改写导致引用失效或内容定位错误。用户在使用时应自行确保链接的准确性与合法性。

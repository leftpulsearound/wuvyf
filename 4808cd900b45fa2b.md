# ResourceHub

ResourceHub 是一个面向开发者、技术研究人员与内容创作者的轻量级技术资源导航与百科类文章聚合平台。该项目定位为外链型知识中台，通过结构化收录高质量的技术解读、百科条目与开发参考文章，帮助用户快速定位所需信息，降低信息检索成本。

ResourceHub 不生产原始内容，而是通过人工筛选与自动化校验机制，将分散在互联网各处的优质技术百科文章统一索引、分类并呈现。项目目标用户包括正在学习编程语言的初学者、需要查阅底层实现细节的工程师、以及需要为技术文档收集参考资料的技术作者。

## 功能概览

百科文章聚合索引：自动抓取并解析来自可信来源的技术百科类文章，提取标题、摘要、发布时间与关键词，供用户统一检索。

多维度分类筛选：按照编程语言、操作系统、网络协议、数据结构与算法、开发工具五大维度对文章进行分类，支持按分类浏览。

全文检索支持：基于标题与摘要构建轻量级倒排索引，支持布尔查询与短语匹配，满足精确检索需求。

收藏与待读清单：用户可标记感兴趣的文章并生成个人待读列表，列表数据持久化存储于本地浏览器。

数据导出为结构化格式：支持将检索结果或收藏列表导出为 JSON 或 CSV 格式，便于二次分析或导入其他工具。

文章去重与失效链接检测：定期对已收录链接进行可用性检查，自动标记失效链接并生成报告，保证资源质量。

暗色主题与阅读模式：提供暗色界面主题与聚焦阅读模式，减少长时间阅读时的视觉疲劳。

## 应用场景

技术新人系统学习某一语言的基础概念时，可通过 ResourceHub 检索该语言相关的百科文章，快速获取多篇不同侧重的解读，从而形成全面的认知，避免遗漏关键知识点。

技术作者在撰写某一技术主题的博客或文档时，可使用 ResourceHub 收集参考资料与引用来源，通过多篇文章交叉验证事实，提高技术写作的准确性与权威性。

开发团队在内部技术选型或方案评审前，可查阅 ResourceHub 中关于特定中间件或协议的文章，快速了解其设计思路与适用边界，为决策提供信息支撑。

## 快速开始

以下步骤帮助您在本地环境中快速启动 ResourceHub 服务。

```bash
# 克隆代码仓库
git clone https://github.com/resourcehub/resourcehub.git
cd resourcehub

# 安装项目依赖
npm install

# 启动开发服务器
npm run dev
```

启动成功后，访问控制台输出的本地地址即可开始使用。生产环境部署请参考文档导航中的部署指南。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.x 或更高 | 是 | 项目运行时环境，推荐使用 LTS 版本 |
| npm 9.x 或更高 | 是 | 包管理器，用于安装项目依赖 |
| SQLite 3 | 是 | 内置元数据存储与索引数据库，无需额外安装 |
| Git 2.x | 是 | 用于克隆代码仓库及版本管理 |
| 现代浏览器（Chrome 110+ / Firefox 110+ / Edge 110+） | 是 | 前端界面运行环境，需支持 ES2022 特性 |
| 网络连接 | 是 | 首次启动需加载部分前端资源及进行文章索引初始化 |
| 磁盘空间 500MB 以上 | 推荐 | 用于存储索引缓存与日志文件 |
| Docker（可选） | 否 | 若使用容器化部署方式则需安装 |

## 常见问题

问：启动时提示 SQLite 数据库连接失败，应如何解决？

答：请检查 data 目录是否存在且具有写入权限。若目录不存在，请手动创建 `data` 目录。若权限不足，请执行 `chmod 755 data`（Linux/macOS）或调整目录安全设置（Windows）。如问题持续，可尝试删除 `data/resourcehub.db` 文件后重新运行 `npm run init-db` 初始化。

问：文章索引抓取失败或超时，是什么原因？

答：索引抓取依赖于网络环境与目标站点的响应速度。请确保网络连接稳定且能够正常访问 `h5.baike.occzl.cn` 域名。若个别链接持续超时，可能是目标站点临时不可用，系统会自动跳过并在日志中记录。您可以手动执行 `node scripts/validate-links.js` 检查全部链接状态。若大量链接失败，请检查防火墙或代理配置。

问：搜索结果为何缺少某些关键词匹配？

答：当前检索基于标题与摘要字段构建的简单倒排索引，暂不支持正文全文检索。若需查找特定术语，建议尝试同义词或更短的关键词组合。项目路线图中计划在未来版本引入基于词干提取和同义词扩展的检索增强功能。

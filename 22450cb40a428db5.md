# IndexWeaver

IndexWeaver 是一个面向技术研究者、开发者与内容策展人的轻量级外链资源归集与导航系统。该项目并不生产原始内容，而是以高度结构化的方式对分散于网络各处的技术文档、百科条目、教程笔记与参考手册进行索引整理，帮助用户在特定技术领域内快速定位高价值信息节点。IndexWeaver 适用于个人知识库的补充检索层、团队内部技术wiki的外部参考延伸，以及开源项目文档站点的关联资源挂载模块。

项目定位为“技术资源的超链接组织工具”，其核心设计目标包括：URL 的批量规范化管理、资源分类标签的自动推断辅助、原始链接的无损存储与展示，以及面向阅读器的清晰目录呈现。IndexWeaver 不依赖JavaScript，完全基于静态Markdown与轻量级后端脚本运行，适合部署于低资源环境或纯静态托管服务。

## 功能概览

**批量原始链接入库**：支持一次性导入上百条外部URL，自动识别协议头与路径结构，保留链接大小写、后缀名及查询参数，确保与源站完全一致。

**分类目录生成**：根据链接所属域名、路径关键词或人工标注的类别标签，自动将资源归入预定义或自定义的分类目录中，便于按主题浏览。

**元数据提取辅助**：对每条链接尝试提取标题、内容摘要与最后修改时间（依赖目标页面的响应头或HTML元标签），作为检索与排序的辅助依据。

**去重与冲突检测**：在新增链接时检查现有库中是否存在相同URL或高度相似的路径，避免重复收录，并提示操作者进行合并或覆盖决策。

**只读展示模式**：生成面向最终读者的静态导航页面，所有链接以纯文本列表形式原样呈现，不做转跳封装或中间页跳转，保证访问透明度。

**状态标记与过滤**：为每条链接标记可用性状态（有效、失效、待验证），支持按状态过滤显示，辅助维护者定期清理或更新资源。

**批量导出与备份**：支持将当前索引库导出为结构化数据文件（JSON或CSV格式），便于迁移、版本控制或与其他工具集成。

## 应用场景

**技术团队内部知识库的外部参考挂载**：开发团队在维护内部技术文档时，常需要引用外部权威资料。IndexWeaver可作为独立模块，将数十至上百条外部百科链接、官方文档入口与社区讨论帖整理为固定章节，嵌入团队wiki或项目README中，使团队成员可一键直达参考源，减少重复搜索时间。

**开源项目文档站点的“相关资源”页**：开源项目的README或文档站通常需要列举依赖项目、同类工具或延伸阅读材料。IndexWeaver可生成一个结构清晰的“资源列表”页面，将链接按类别（如官方文档、社区教程、视频讲解、问题追踪）分组展示，且所有链接均为原始直链，符合开源社区对透明引用的偏好。

**个人技术博客的“书签墙”**：技术博主或内容创作者可将长期积累的参考链接通过IndexWeaver整理为公开的“书签墙”页面，挂载于博客侧栏或独立子页面。读者可借此了解博主的技术关注领域，同时获得高质量的外部学习入口，提升博客的实用价值。

**技术课程或培训材料的配套导航**：讲师或培训师在分发课程材料时，可将课件中提及的所有外部链接（如论文原文、API手册、在线工具）通过IndexWeaver统一整理为一份附录文档，学员无需手动从PPT中复制粘贴URL，直接访问该导航页即可获取全部资源。

## 快速开始

以下命令演示了如何获取IndexWeaver源码、安装基础依赖并启动本地预览服务。

```bash
git clone https://github.com/indexweaver/indexweaver.git
cd indexweaver
pip install -r requirements.txt
python app.py --scan ./data/links.txt --output ./docs/index.md
```

## 项目结构

```
indexweaver/
├── app.py                     # 主入口脚本，负责扫描、解析与生成输出
├── config.yaml                # 项目配置文件，定义分类规则、输出路径等
├── requirements.txt           # Python依赖列表
├── data/                      # 数据存储目录
│   ├── links.txt              # 原始链接列表（每行一条URL）
│   ├── metadata.db            # SQLite数据库，缓存链接元数据（标题、摘要等）
│   └── batches/               # 按批次归档的原始链接文件
│       └── batch_49.txt       # 第49批次链接列表
├── src/                       # 核心源码目录
│   ├── parser.py              # URL解析与规范化模块
│   ├── fetcher.py             # HTTP请求与元数据抓取模块
│   ├── deduper.py             # 链接去重与冲突检测逻辑
│   ├── renderer.py            # Markdown输出渲染引擎
│   └── utils.py               # 通用工具函数（日志、文件操作等）
├── docs/                      # 生成的文档输出目录
│   ├── index.md               # 主导航页（本文件即由该流程生成）
│   └── resources.md           # 完整资源列表页
├── tests/                     # 单元测试目录
│   ├── test_parser.py
│   ├── test_fetcher.py
│   └── test_deduper.py
├── scripts/                   # 辅助运维脚本
│   ├── verify_links.sh        # 批量验证链接可用性的shell脚本
│   └── export_csv.py          # 导出索引库为CSV格式
└── README.md                  # 项目说明文件（本文件）
```

## 常见问题

**Q：IndexWeaver是否会对原始链接的内容进行缓存或代理？**

A：不会。IndexWeaver仅存储链接本身及其基础的元数据（如页面标题、摘要文本），不会缓存完整的页面内容或资源文件。所有链接在展示时均为原始直链，用户点击后将直接访问目标站点，不经过任何中间代理或跳转页面。这保证了内容来源的透明性与版权合规性。

**Q：如果目标站点无法访问或链接失效，IndexWeaver会如何处理？**

A：IndexWeaver提供了可选的链接状态验证功能（通过 --validate 参数触发）。在验证过程中，系统会发送HEAD请求检查链接的可达性，并将状态标记（有效/失效）写入元数据库。在生成的导航页中，失效链接会以特殊样式标注（如删除线或颜色提示），但链接本身不会被移除，以保留原始引用记录。维护者可定期运行验证脚本并手动更新或移除失效链接。

**Q：IndexWeaver能否处理需要登录或带有反爬机制的网站链接？**

A：IndexWeaver的元数据抓取模块仅使用标准的HTTP请求头（User-Agent为 IndexWeaver/1.0），不执行JavaScript或模拟登录行为。对于需要登录或严格反爬的站点，系统可能无法获取页面标题与摘要，但链接本身仍会正常收录与展示。此类情况下，元数据字段将保持为空，不影响链接的导航功能。建议对于此类资源，在链接旁手动添加备注说明。

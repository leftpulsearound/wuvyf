# ResourceBridge

ResourceBridge 是一个面向技术研究者、内容聚合者与知识管理者的外链资源汇总与导航工具。该项目不生产内容，而是通过结构化的方式将分散在网络各处的优质技术文章、百科条目、开发文档与参考链接进行统一收集、分类与呈现，帮助用户在海量信息中快速定位到真正有价值的阅读材料。

ResourceBridge 的目标用户包括正在学习新技术栈的开发者、需要持续跟进技术动态的架构师、撰写技术博客或论文的研究人员，以及任何希望建立个人知识索引体系的互联网从业者。项目本身不依赖复杂后端，以纯静态资源组织形式交付，可轻松部署到任意 Web 服务器或对象存储服务中。

## 功能概览

**批量外链导入与解析**：支持从外部数据源批量导入 URL 列表，自动识别链接类型并提取标题信息，减少手动整理的成本。

**多维度分类索引**：所有收录链接可按主题领域、内容类型、来源站点等多个维度进行自由分类，支持一链多标签的灵活标记体系。

**全文检索与快速过滤**：内置基于标题和摘要的轻量级检索功能，允许用户通过关键词、分类标签或发布日期快速筛选所需资源。

**阅读状态跟踪**：为每个资源链接提供“未读”、“已读”、“收藏”三种状态标记，方便用户管理个人阅读进度与兴趣清单。

**数据导入导出**：支持 JSON、CSV、Markdown 表格三种格式的数据导入导出，便于与其他知识管理工具（如 Notion、Obsidian）进行数据交换。

**响应式布局与移动适配**：前端界面针对桌面端与移动端分别进行优化，确保在手机、平板与电脑上均能获得良好的浏览体验。

**外部资源自动快照**：对收录的每一篇外链文章自动抓取页面标题、发布时间与摘要描述，形成本地元数据缓存，即使原链接失效也能保留基本参考信息。

## 应用场景

**个人技术阅读清单管理**：开发者可将日常浏览过程中发现的优质技术博客、官方文档、API 参考链接统一存入 ResourceBridge，按技术栈或学习阶段分类，形成专属的成长路线图。每周固定时间回顾清单，系统性地完成阅读与消化。

**团队知识库外链整合**：技术团队可将组内共享的第三方参考资料、竞品分析报告、行业白皮书链接集中收录于 ResourceBridge 实例中，部署在内网服务器上供全体成员访问。新人入职时可快速浏览团队沉淀的外部知识体系，缩短上手周期。

**技术社区内容推荐系统原型**：社区运营者可将历史发布过的推荐阅读链接导入 ResourceBridge，按主题或热度重新组织编排，生成面向不同读者群体的推荐列表，作为社区内容分发的一个轻量化辅助渠道。

**学术文献外链索引**：研究人员在撰写文献综述时，可将散落在各期刊网站、预印本平台、机构知识库中的论文链接统一采集至 ResourceBridge，添加研究主题标签与阅读优先级，构建可检索的个人文献库。

## 快速开始

以下步骤帮助您在本地环境快速启动 ResourceBridge 服务。

```bash
# 克隆项目仓库
git clone https://github.com/resourcebridge/resourcebridge.git

# 进入项目目录
cd resourcebridge

# 安装依赖（基于 Node.js 与 npm）
npm install

# 启动开发服务器
npm run dev
```

执行上述命令后，打开浏览器访问 http://localhost:3000 即可开始使用。生产环境部署请参考 `docs/deployment.md` 中的说明。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js >= 18.0.0 | 是 | 项目运行时环境，推荐使用 LTS 版本 |
| npm >= 9.0.0 或 yarn >= 1.22.0 | 是 | 包管理器，用于安装项目依赖 |
| SQLite 3 | 是 | 内置数据存储引擎，无需额外安装，项目启动时自动初始化 |
| 现代浏览器（Chrome / Firefox / Safari / Edge 最新两个版本） | 是 | 前端界面运行环境，需支持 ES2020 与 CSS Grid |
| Git >= 2.30.0 | 否 | 仅当需要从源码构建或参与贡献时需要 |
| Docker >= 20.10.0 | 否 | 可选，用于容器化部署 |

## 项目结构

```
resourcebridge/
├── backend/                          # 后端服务代码
│   ├── src/
│   │   ├── controllers/              # 请求控制器，处理路由逻辑
│   │   ├── models/                   # 数据模型定义（链接、分类、用户状态）
│   │   ├── services/                 # 业务服务层（导入、快照、检索）
│   │   ├── middleware/               # 认证、日志、跨域中间件
│   │   └── utils/                    # 通用工具函数（URL 解析、日期格式化）
│   ├── routes/                       # API 路由定义
│   ├── migrations/                   # 数据库迁移脚本
│   └── tests/                        # 后端单元测试与集成测试
├── frontend/                         # 前端界面代码
│   ├── src/
│   │   ├── pages/                    # 页面级组件（首页、列表页、详情页）
│   │   ├── components/               # 可复用 UI 组件（导航栏、卡片、搜索框）
│   │   ├── stores/                   # 状态管理（Pinia / Vuex 模块）
│   │   ├── utils/                    # 前端工具函数（日期格式化、文本截断）
│   │   └── styles/                   # 全局样式与主题变量
│   ├── public/                       # 静态资源（favicon、robots.txt）
│   └── tests/                        # 前端组件测试
├── docs/                             # 项目文档
│   ├── user-guide/                   # 用户手册
│   ├── deployment/                   # 部署指南
│   ├── development/                  # 开发文档
│   └── operations/                   # 运维手册
├── scripts/                          # 辅助脚本（数据迁移、批量导入、备份）
├── data/                             # 本地数据存储目录（SQLite 数据库文件）
├── logs/                             # 运行日志目录
├── .env.example                      # 环境变量配置模板

├── package.json                      # 项目依赖声明
├── README.md                         # 项目说明文档（本文件）
└── LICENSE                           # MIT 许可证文件
```

## 常见问题

**Q：ResourceBridge 是否支持在线直接打开外链而不跳转？**

A：项目本身不代理或改写外链内容，所有链接点击后均直接跳转至原始地址。但前端界面会通过元数据缓存展示标题、摘要等信息，方便您在不离开列表页的情况下预览内容概要。如需完整阅读，请点击链接访问原始页面。

**Q：导入大量链接后页面加载变慢，如何优化？**

A：当收录链接数量超过 2000 条时，建议在 `backend/src/services/` 中启用分页查询（默认每页 50 条），并确保数据库中的 `links` 表已为 `category` 和 `created_at` 字段建立索引。您也可以定期运行 `scripts/cleanup-duplicates.js` 清理重复条目，并关闭不必要的元数据快照功能以减轻存储压力。

**Q：能否将 ResourceBridge 部署到没有 Node.js 环境的共享主机？**

A：可以。项目支持输出纯静态文件模式，执行 `npm run build:static` 后，所有页面将预渲染为 HTML 文件存放在 `dist/` 目录中，您可以将该目录直接上传至任何支持静态文件托管的 Web 服务器或 CDN 服务。但需要注意，静态模式下检索、状态跟踪等动态功能将不可用，仅适用于只读展示场景。

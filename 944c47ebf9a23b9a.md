# LinkVault

LinkVault 是一个面向技术研究与知识管理场景的轻量级外链资源聚合与导航系统。该项目定位于帮助开发者、技术文档撰写者、运维工程师以及数据分析人员快速检索、分类管理和复用分散于互联网各处的优质百科类、技术参考类与数据类链接资源。LinkVault 不提供内容存储服务，而是通过结构化的索引机制与清晰的目录划分，将原本碎片化的外链资源转化为可维护、可共享、可版本控制的知识资产。本项目特别适用于需要批量处理外部参考资料链接、构建内部技术知识库或搭建团队共享书签系统的用户群体。

## 功能概览

**批量链接导入与解析** 支持从纯文本、CSV 或 JSON 格式批量导入 URL 列表，自动识别链接协议与路径结构，并完成基础的可用性探测。

**多维度分类标签系统** 允许用户为每条链接资源赋予自定义标签（如“百科”、“API文档”、“运维手册”），支持多标签组合筛选与快速定位。

**链接状态健康检查** 内置定时任务，可对已收录的链接进行周期性的 HTTP 状态码检测，标记失效或重定向的链接，辅助用户清理或更新资源。

**全文检索与高级过滤** 基于链接标题、描述、标签以及 URL 路径关键词构建倒排索引，支持布尔逻辑检索与通配符匹配，提升大规模链接库下的查找效率。

**数据导入导出接口** 提供 RESTful API 与命令行工具，支持将链接库导出为 Markdown 表格、JSON 或 HTML 书签文件，便于与其他知识管理工具集成。

**用户自定义视图** 支持按“最近访问”、“新增时间”、“域名分组”等多种排序维度，用户可以保存常用筛选条件为自定义视图，实现个性化资源浏览体验。

## 应用场景

**技术团队内部知识库构建** 开发团队在项目迭代过程中会产生大量参考链接（如依赖库官方文档、故障排查社区讨论、性能优化案例）。LinkVault 可以将这些链接统一收录，按业务模块打标签，并定期检查链接可用性，确保知识库的长期有效性。

**开源项目文档外链管理** 开源项目维护者在编写 README 或 Wiki 时，需要引用大量外部资料。LinkVault 可作为外链的中转索引，维护者先在本系统内整理好分类与描述，再通过导出功能生成结构化的链接清单，直接嵌入项目文档，避免硬编码 URL 散落各处。

**技术写作与内容策展** 技术博主或教育内容创作者在筹备系列教程时，需收集大量背景阅读材料。借助 LinkVault 的标签与检索功能，可以快速按主题（如“Kubernetes”、“Python异步编程”）重组链接资源，辅助内容大纲的构思与素材编排。

**运维故障排查参考手册** 运维工程师可将常见错误码对应的解决方案链接、官方公告页、社区 issue 讨论地址等集中收录，并结合健康检查功能，当某参考链接失效时及时收到提醒，避免在紧急故障处理时访问无效页面。

## 快速开始

以下命令演示了如何从 GitHub 克隆 LinkVault 源码、安装项目依赖并启动开发服务器。

```bash
git clone https://github.com/linkvault/linkvault.git
cd linkvault
pip install -r requirements.txt
python manage.py runserver
```

上述步骤完成后，访问本地 8000 端口即可进入 LinkVault 管理界面。生产环境部署请参考 `docs/deployment.md` 中的配置说明。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.8 及以上 | 项目核心运行环境，用于驱动后端服务与 CLI 工具 |
| Django | 3.2 LTS | Web 框架，负责路由、ORM 及管理后台功能 |
| Celery | 5.2.0 | 分布式任务队列，用于调度链接健康检查等异步任务 |
| Redis | 6.0 及以上 | 作为 Celery 的消息代理（Broker）和结果后端（Backend） |
| PostgreSQL | 12.0 及以上 | 主数据库，存储链接元数据、标签体系及用户配置 |
| Nginx | 1.18 及以上 | 生产环境推荐的反向代理服务器，用于处理静态文件与负载均衡 |

## 项目结构

```
linkvault/
├── cmd/                                  # 命令行入口与 CLI 工具实现
│   ├── server/                           # 启动 Web 服务的命令模块
│   └── cli/                              # 数据导入导出、健康检查等运维命令
├── config/                               # 全局配置文件目录
│   ├── settings.py                       # Django 基础配置（数据库、中间件、静态路径）
│   ├── celery.py                         # Celery 任务队列配置与周期性任务注册
│   └── urls.py                           # 根路由映射，包含 API 与后台管理路由
├── apps/                                 # 核心业务应用模块
│   ├── links/                            # 链接资源管理应用：模型、视图、序列化器
│   ├── tags/                             # 标签体系应用：标签树结构、关联逻辑
│   ├── checks/                           # 健康检查模块：HTTP 状态探测、告警规则
│   └── users/                            # 用户认证与个人配置管理
├── static/                               # 前端静态资源（CSS、JavaScript、字体文件）
│   ├── css/                              # 基于 Bootstrap 5 定制的基础样式
│   └── js/                               # 前端交互逻辑：搜索过滤、标签编辑、视图切换
├── templates/                            # Django 模板文件，包含后台管理界面骨架
│   ├── base.html                         # 全局基础模板，包含导航栏与页脚
│   └── links/                            # 链接列表、详情、导入页面的专用模板
├── tests/                                # 单元测试与集成测试用例
│   ├── test_models.py                    # 数据模型层的测试
│   └── test_api.py                       # RESTful API 接口的功能测试
```

## 常见问题

**问：LinkVault 能否直接存储链接对应的网页内容副本或快照？**

答：不能。LinkVault 的设计原则是仅存储链接元数据（URL、标题、标签、描述）和状态信息，不爬取或存储页面内容。用户如需持久化页面快照，建议搭配第三方工具（如 ArchiveBox）使用，LinkVault 可为其提供索引入口。

**问：健康检查功能是否会因为请求频率过高而被目标网站封禁？**

答：健康检查默认采用单线程顺序探测，且每次请求之间设有 2 秒的延迟间隔。用户可在配置文件中调整 `CHECK_INTERVAL` 与 `TIMEOUT` 参数以控制探测频率和超时时间。对于高频访问场景，建议配置代理池或降低检查周期。

**问：如何从现有浏览器书签或收藏夹迁移数据到 LinkVault？**

答：LinkVault 提供了 `cli/import` 子命令，支持解析 Netscape 格式（HTML）的书签导出文件，以及 Firefox/Chrome 的 JSON 格式书签备份。用户只需执行 `python manage.py import_bookmarks --file bookmarks.html` 即可完成批量迁移，系统会自动尝试解析书签文件夹结构并转换为标签。
